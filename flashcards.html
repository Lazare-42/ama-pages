1. Mon écran — qu'est-ce que je vois ?

01 L'écran est coupé en plusieurs parties

Quand tu te connectes, tu arrives sur tmux — un outil qui découpe ton écran en plusieurs zones. Voici ce que tu vois :

┌─────────────────────────────────────────────────────────────────────────────────────────┌ │ │ DROITE HAUT │ │ GAUCHE (grande) │ Shell normal │ │ │ (listing de fichiers, commandes) │ │ CLAUDE CODE │ Tu n'en as pas besoin │ │ ├─────────────────────────────────┤ │ C'est ICI que tu me parles │ DROITE BAS │ │ et que tu lis mes réponses │ Shell normal │ │ │ (git, commandes de Laz) │ │ Tu tapes en bas de cette zone │ Tu n'en as pas besoin │ │ │ │ └─────────────────────────────────────────────────────────────────────────────────────────┘
La partie gauche (la plus grande) = Claude Code = là où on discute. Les 2 parties à droite sont des terminaux normaux — et tu peux les utiliser (voir fiche 05) !
02 Les panneaux de droite — ton explorateur de fichiers

Les 2 zones à droite ne sont pas juste décoratives ! Tu peux les utiliser comme un explorateur de fichiers. Pour y aller :

Ctrl+B  puis  flèche droite    (aller dans le panneau de droite)
Ctrl+B  puis  flèche gauche    (revenir sur Claude Code)

Une fois dans un panneau de droite, tape :

cd /run/media/lazrossi/KINGSTON/code/yann/AMA
ls

Et tu verras tous tes dossiers et fichiers AMA. Tu peux naviguer :

ls              → voir le contenu du dossier actuel
ls Docs/        → voir le contenu de Docs/
ls Portfolios/  → voir le contenu de Portfolios/
ls Site/        → voir le contenu de Site/

04 C'est quoi Claude Code ?

C'est un assistant dans ton terminal. Tu lui parles en français, il comprend, et il peut :

  • Lire tes fichiers (HTML, docs, images...)
  • Créer et modifier des fichiers
  • Envoyer des fichiers en ligne (0x0.st)
  • Chercher des infos dans ton projet
  • Organiser, trier, résumer

C'est comme un collègue qui a les mains sur le clavier — il peut agir, pas juste parler.

Différence avec claude.ai (le chat web) : claude.ai peut seulement discuter. Claude Code peut aussi toucher à tes fichiers directement.
05 L'interface Claude Code elle-même

Dans la zone gauche (Claude Code), voici ce que tu vois :

  • En haut : mes réponses qui défilent (texte, tableaux, code...)
  • En bas : la zone où tu tapes ton message (avec un curseur qui clignote)
  • Des blocs colorés qui apparaissent : ce sont mes actions (lire un fichier, modifier, exécuter une commande)

Quand je te demande une permission (modifier un fichier, lancer une commande), tu vois un message avec des options. Tu peux accepter ou refuser.

Tu peux sélectionner du texte avec la souris dans le terminal — pratique pour copier un lien ou un bout de réponse.
06 Je peux scroller pour relire ?

Oui ! Si la conversation est longue et que tu veux remonter :

  • Molette de la souris → scroller dans la conversation
  • Si ça ne marche pas, c'est peut-être tmux qui bloque : essaie Ctrl+B puis [ pour entrer en mode « scroll », et q pour en sortir

2. Me connecter — la séquence complète

07 La procédure de connexion pas à pas

Depuis ton Beelink (Windows 11) :

1

Ouvrir le terminal

Cherche « Terminal » dans la barre de recherche Windows, ou tape cmd

2

Se connecter au serveur de Laz

Tape la commande SSH que Laz t'a donnée + mot de passe

3

Retrouver ta session

tmux attach

Si ça dit « no sessions » ou que tu arrives sur la mauvaise session :

tmux list-sessions      (voir les sessions dispo)
tmux attach -t 0        (se connecter à la session 0)
4

Naviguer entre les sessions tmux

Ctrl+B  puis  S

Un menu s'affiche avec les sessions disponibles. Utilise les flèches pour choisir la tienne, puis Entrée.

5

Tu es sur Claude Code

Tu vois l'écran divisé (voir section 1). La partie gauche = Claude Code. Tu peux commencer à discuter.

08 Si Claude Code n'est pas lancé

Si la partie gauche montre juste un terminal normal (pas de conversation), il faut lancer Claude Code :

cd /run/media/lazrossi/KINGSTON/code/yann/AMA
claude
Le dossier où tu lances claude est important — c'est son « contexte ». Il voit les fichiers de ce dossier. Toujours lancer depuis le dossier AMA.

3. Parler à Claude — les bases

09 Comment je lui parle ?

Tu écris en français, comme un message à un collègue. Pas besoin de code, pas besoin de syntaxe spéciale.

« salut, on met à jour la conciergerie »
« crée-moi une fiche pour Rougerie »
« uploade le dashboard sur 0x0.st »
Plus tu es précis, mieux ça marche. « mets à jour la conciergerie : ménage S1 fait, serviettes S2 manquent » > « mets à jour le truc »
10 Les permissions — quand Claude te demande « OK ? »

Avant de faire quelque chose d'important, Claude te demande la permission :

  • Modifier un fichier → il te montre ce qui va changer (en vert/rouge)
  • Lancer une commande → il te montre la commande avant de l'exécuter

Tu peux :

  • Accepter (Entrée ou « y ») → il fait l'action
  • Refuser (n) → il ne fait rien et te propose autre chose
Prends l'habitude de lire ce que Claude propose avant de dire oui. Surtout pour les commandes git push, rm, ou tout ce qui envoie des données quelque part.
11 Les commandes spéciales (avec /)

Ce sont les seules « commandes » à connaître. Tu les tapes directement dans Claude Code :

/clear     → Recommencer une conversation fraîche
              (utile si tu changes de sujet)

/compact   → Compresser la conversation
              (quand elle devient très longue)

/cost      → Voir combien la session a coûté

/help      → Aide générale
/clear est ton meilleur ami. Quand tu sens que Claude mélange les sujets ou que la session devient confuse → /clear et on repart propre.

Les fichiers HTML d'AMA ne sont pas sauvés automatiquement (pas de git). Bonne habitude :

Si tu fais une erreur et que la session est fermée, sans backup c'est perdu. Fais-le au moins avant les grosses sessions.

6. Apprendre petit à petit

19 Tu n'as pas besoin de tout savoir

Claude Code fait le travail technique pour toi. Ton rôle c'est de :

  • Savoir ce que tu veux (mettre à jour X, créer Y)
  • Vérifier ce que Claude propose (lire les diffs)
  • Dire non quand c'est pas ce que tu veux

Le reste, tu l'apprends au fil des sessions. Pas besoin de cours théoriques.

20 Niveau 1 — Discuter et organiser

C'est là où tu en es, et c'est déjà énorme. Tu peux :

  • Demander à Claude de lire, modifier, créer des fichiers
  • Organiser tes idées et structurer tes projets
  • Uploader et partager des fichiers
  • Tenir à jour tes docs (conciergerie, fiches, liens...)
C'est exactement ce qu'on fait depuis 2-3 semaines. Tu n'as pas besoin de « coder » pour être productif.
21 Niveau 2 — Comprendre ce que Claude fait

Au fil des sessions, tu commences à reconnaître des patterns :

  • Les commandes curl pour uploader
  • La structure HTML (<div>, <p>, <a href="...">)
  • Les fichiers Markdown (.md) pour la doc
  • Le diff vert/rouge quand Claude modifie un fichier

Tu peux demander à Claude d'expliquer :

« explique-moi ce que tu viens de faire »
« c'est quoi cette commande curl ? »
« pourquoi tu as ajouté cette ligne ? »
22 Niveau 3 — Prendre la main

À terme, tu pourras :

  • Avoir ton propre compte Anthropic (Claude Code)
  • Lancer Claude Code toi-même depuis ton propre terminal
  • Gérer tes fichiers sans passer par la machine de Laz
  • Utiliser les scripts (backup, start) en autonomie
Pas de pression — c'est un objectif, pas un prérequis. Chaque session t'apprend un peu plus.

7. Quand ça merde

23 Claude part dans la mauvaise direction
  • Ctrl+C pour l'arrêter net
  • Dire « stop, c'est pas ce que je veux »
  • Si c'est le chaos : taper /clear et recommencer
C'est normal et ça arrive. Claude ne se vexe pas. Tu peux dire « non », « annule », « recommence autrement ».
24 Claude a modifié un fichier par erreur
  • Pendant la session : dis « remets le fichier comme avant » (il s'en souvient)
  • Session terminée : restaurer depuis le dernier backup
C'est pour ça qu'on fait un backup en début de session !
25 Erreur 451 sur 0x0.st

Le site 0x0.st bloque parfois les uploads (fichier déjà connu).

« ajoute un commentaire pour changer le hash et re-uploade »

Claude sait gérer — il ajoute un petit texte invisible dans le fichier et retente.

26 L'écran a l'air bug ou gelé
  • Claude réfléchit : parfois il met du temps (surtout pour les gros fichiers). Attends quelques secondes.
  • Tmux a gelé : tape Ctrl+C. Si rien ne se passe, essaie Ctrl+B puis : puis tape kill-pane (attention : ça ferme le panneau)
  • Connexion coupée : reconnecte-toi (SSH + tmux attach). Ta session tmux est toujours là, elle ne se perd pas.
Tmux garde ta session même si tu perds la connexion internet. Quand tu te reconnectes, tout est toujours là.
27 La clé USB n'est pas détectée

Si erreur « no such file or directory » pour /run/media/lazrossi/KINGSTON/ :

  • La clé USB KINGSTON n'est pas branchée sur la machine de Laz
  • Prévenir Laz pour qu'il la branche
Tout AMA vit sur la clé USB. C'est branché sur la machine de Laz, pas sur ton Beelink. Si elle n'est pas là, il faut que Laz la branche.
